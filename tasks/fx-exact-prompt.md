# ePulse 환율 상세 학습 뷰 구현 프롬프트

당신은 React 기반 정적 웹앱을 구현하는 시니어 프론트엔드 엔지니어다.  
이 작업은 ePulse의 **환율 상세 학습 뷰**를 구현하는 것이다.

---

## 1. 목표

거시 인과관계 메인 화면에서 사용자가 **원/달러 환율 노드**를 클릭하면,  
현재 거시 상태를 전달받아 **환율이 어떻게 결정되고 왜 금리, 자금 이동, 수출입 요인에 반응하는지**를 직관적으로 이해할 수 있는 상세 화면을 구현하라.

이 화면은:
- 환율 개념을 모르는 사용자도 이해 가능해야 하며
- 환율을 단순 숫자가 아니라 **달러 수요와 공급이 만든 가격**으로 이해하게 해야 한다
- 미국 금리, 한국 금리, 성장률, 외국인 자금, 유가가 환율에 어떻게 연결되는지 보여야 한다

이 화면은 **실전 FX 트레이딩 도구가 아니라 교육용 인과관계 시뮬레이터**다.

---

## 2. 핵심 UX 개념

사용자는 이 화면을 보고 아래를 이해해야 한다.

- 환율은 어떻게 결정되는가
- 왜 미국 금리가 오르면 원/달러 환율이 오를 수 있는가
- 왜 외국인 자금이 빠져나가면 환율이 오를 수 있는가
- 왜 유가 상승이 환율 상승 압력으로 이어질 수 있는가
- 왜 한국 성장률 개선이 환율 상승을 완화할 수 있는가

핵심은 **환율 = 달러를 사고파는 힘의 균형 결과**라는 점을 시각적으로 전달하는 것이다.

---

## 3. 상위 화면과의 연결

### 진입 방식
- 거시 화면에서 `원/달러 환율` 또는 `한국 환율` 관련 노드 클릭
- 현재 macro state를 상세 화면으로 전달
- 전달 방식은 router state 또는 query-serializable state 사용

### 복귀 방식
- 상단에 `← 거시 화면으로 돌아가기`
- 복귀 시 원래 거시 상태 유지

### 자연스러운 연결
- 메인 화면과 동일한 시각 언어 유지
- 다크 테마, 카드 기반 노드, 연결선 강조, 설명 패널 유지

## 4. 화면의 핵심 개념 구조

### 1층: 입력 요인
- 한국 기준금리
- 미국 기준금리
- 한국 성장률
- 외국인 자금
- 유가

### 2층: 내부 결정 구조
- 달러 수요
- 달러 공급
- 원화 매력도
- 달러 매력도

### 3층: 결과
- 원/달러 환율

## 6. 핵심 인과관계

### 6.1 입력 → 내부 구조

#### 한국 기준금리
- 한국 기준금리 상승 → 원화 매력도 상승

#### 미국 기준금리
- 미국 기준금리 상승 → 달러 매력도 상승
- 미국 기준금리 상승 → 달러 수요 증가 압력

#### 한국 성장률
- 한국 성장률 상승 → 원화 매력도 상승
- 한국 성장률 상승 → 달러 공급 개선에 약한 기여 가능

#### 외국인 자금
- 외국인 자금 유입 → 달러 공급 증가 + 원화 수요 증가
- 외국인 자금 유출 → 달러 수요 증가

#### 유가
- 유가 상승 → 수입 비용 증가 → 달러 수요 증가

### 6.2 내부 구조 간 관계

- 원화 매력도 상승 → 달러 수요를 완화
- 달러 매력도 상승 → 달러 수요를 자극
- 외국인 자금 유입은 원화 매력도에도 간접적으로 긍정적 신호를 줄 수 있음

### 6.3 내부 구조 → 결과

- 달러 수요 상승 → 원/달러 환율 상승 압력
- 달러 공급 상승 → 원/달러 환율 하락 압력
- 원화 매력도 상승 → 원/달러 환율 하락 압력
- 달러 매력도 상승 → 원/달러 환율 상승 압력

## 7. 계산 모델

현실을 정밀 복제하지 말고, **설명 가능한 단순 가중치 모델**을 사용하라.

모든 입력값은 0~100 범위이며, 50이 중립이다.

### 7.1 내부 노드 계산

#### 달러 수요
dollarDemand =
- `+ 0.45 * (us_rate - 50)`
- `- 0.40 * (foreign_flow - 50)`
- `+ 0.30 * (oil - 50)`

해석:
- 미국 금리가 오를수록 달러를 들고 있으려는 유인이 커진다
- 외국인 자금이 한국으로 들어오면 달러 수요는 줄어든다
- 유가가 오르면 수입 대금 지급용 달러 수요가 늘어난다

#### 달러 공급
dollarSupply =
- `+ 0.45 * (foreign_flow - 50)`
- `+ 0.15 * (kr_growth - 50)`

해석:
- 외국인 자금 유입은 달러 공급을 늘린다
- 성장률 개선은 수출 및 자금 유입 기대를 통해 달러 공급 개선에 약한 기여를 할 수 있다

#### 원화 매력도
krwAttractiveness =
- `+ 0.45 * (kr_rate - 50)`
- `+ 0.35 * (kr_growth - 50)`

해석:
- 한국 금리가 높을수록 원화 보유 매력이 높아진다
- 한국 성장률이 높을수록 한국 자산 선호가 높아질 수 있다

#### 달러 매력도
usdAttractiveness =
- `+ 0.55 * (us_rate - 50)`
- `- 0.10 * (kr_rate - 50)`

해석:
- 미국 금리가 높을수록 달러 매력도가 커진다
- 한국 금리가 상대적으로 높아지면 달러 매력도는 일부 완화된다

### 7.2 결과 노드 계산

#### 원/달러 환율
usdkrwResult =
- `+ 0.40 * dollarDemand`
- `- 0.30 * dollarSupply`
- `- 0.30 * krwAttractiveness`
- `+ 0.30 * usdAttractiveness`

모든 결과는 `-100 ~ +100` 범위로 clamp 하라.

## 8. 상태 라벨링

숫자 자체보다 **상태 언어**로 보여줘야 한다.

### 공통 상태 라벨
- 강한 상승 압력
- 상승 압력
- 중립
- 하락 압력
- 강한 하락 압력

### 내부 노드용 라벨 예시

#### 달러 수요 / 달러 공급
- 매우 강함
- 강함
- 보통
- 약함
- 매우 약함

#### 원화 매력도 / 달러 매력도
- 매우 높음
- 높음
- 중립
- 낮음
- 매우 낮음

#### 환율 결과
- 강한 원화 약세 압력
- 원화 약세 압력
- 중립
- 원화 강세 압력
- 강한 원화 강세 압력

주의:  
`원/달러 환율 상승 = 원화 약세`  
`원/달러 환율 하락 = 원화 강세`  
이 해석이 UI 전반에서 일관되어야 한다.

## 9. 핵심 시각화 요구사항

### 가장 중요한 표현
이 화면은 반드시 아래를 보여줘야 한다.

- 달러를 사려는 힘이 커지면 환율이 오른다
- 달러를 파는 힘이 커지면 환율이 내린다
- 미국 금리, 한국 금리, 외국인 자금, 유가가 서로 다른 방식으로 환율에 기여한다

즉, 환율을 **추상적인 숫자**가 아니라  
**달러 수요/공급과 상대적 통화 매력도의 결과**로 느끼게 해야 한다.

## 10. UI 레이아웃

### 좌측 패널
입력 요인 카드 목록:
- 한국 기준금리
- 미국 기준금리
- 한국 성장률

각 카드에는:

- 현재 상태

- 값 강도 표시

읽기 전용으로 시작해도 되지만,

### 중앙 캔버스
노드 기반 구조 시각화:
- 입력 노드 → 내부 구조 노드 → 결과 노드
- 활성 경로는 밝고 굵게
- 약한 경로는 흐리게
- hover 시 관련 경로만 강조

### 오른쪽 패널
설명 패널:
- 현재 환율 구조 요약
- 환율 상승/하락의 주요 원인
- 왜 그런 결과가 나왔는지 자연어 설명
- 즉시 / 단기 / 중기 흐름 설명

## 12. 설명 패널 문장 생성 규칙

설명은 룰 기반 자연어 생성으로 구현하라.  
LLM 호출은 사용하지 않는다.

### 반드시 포함할 설명
- 현재 환율 압력 요약
- 주요 상승 요인 2개
- 주요 하락 요인 2개
- 즉시 / 단기 / 중기 해석
- 원화 강세/약세 의미 설명

### 예시 문장
- “미국 기준금리 상승으로 달러 매력도가 높아지며 원/달러 환율 상승 압력이 커지고 있습니다.”
- “반면 한국 성장률 개선과 한국 금리 상승은 원화 매력도를 높여 환율 상승을 일부 완화하고 있습니다.”
- “외국인 자금 유입은 달러 공급을 늘리고 원화 수요를 자극해 환율 하락 압력으로 작용할 수 있습니다.”
- “유가 상승은 수입 대금 결제를 위한 달러 수요를 늘려 환율 상승 압력에 기여합니다.”

## 13. 시각 스타일

아름다운 비주얼은 매우 중요하다.  
이 도구는 단순 교육용 화면이 아니라 **탐색하고 싶은 감각적인 학습 도구**여야 한다.

### 스타일 가이드
- 다크 배경
- 깊이감 있는 카드
- 선명하지만 과하지 않은 글로우
- 부드러운 애니메이션
- 미니멀한 정보 밀도

### 색상 원칙
- 환율 상승 압력 / 달러 강세 / 달러 수요 강함: 주황~빨강 계열
- 환율 하락 압력 / 원화 강세 / 달러 공급 강함: 청록~파랑 계열
- 중립: 회색/중성톤
- 의미가 같은 색상은 앱 전체에서 일관 유지

### 노드 카드
각 노드는 다음을 포함:
- 제목
- 상태 라벨
- 아이콘
- 미니 강도 표시 바 또는 원형 인디케이터
- 지역 태그 (KR / US / GL 필요 시)

## 14. 인터랙션

### hover
- 노드 hover 시 관련 경로 강조
- 우측 설명 패널에 해당 노드 설명 강조 가능

### click
- 결과 노드 클릭 시 관련 원인 경로 전체 강조
- 예: `원/달러 환율` 클릭 시
  - 달러 수요
  - 달러 공급
  - 원화 매력도
  - 달러 매력도
  경로가 더 밝게 보이도록 처리

### animation
- 값 변화 시 노드 상태 부드럽게 전환
- 연결선 강조 시 은은한 흐름 효과
- 라인 차트 부드럽게 재계산

## 19. 설계 원칙

선택이 필요할 때는 아래를 우선하라.

1. 정확성보다 설명력
2. 복잡성보다 직관성
3. 숫자보다 관계
4. 결과보다 과정
5. 정보량보다 시각적 명확성

이 원칙을 기준으로 완성도 높은 UI와 교육용 인과관계 경험을 구현하라.
