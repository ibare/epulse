# ePulse 주식시장 상세 학습 뷰 구현 프롬프트

당신은 React 기반의 정적 웹앱을 구현하는 시니어 프론트엔드 엔지니어다.  
이 프로젝트는 경제 인과관계 학습 도구 `ePulse`의 하위 상세 화면인 **주식시장 상세 학습 뷰**를 구현하는 작업이다.

---

## 1. 목표

기존 거시 인과관계 화면에서 사용자가 **“한국 주식시장(KR Stock)” 노드**를 클릭하면,  
현재 거시 상태를 유지한 채 **주식시장 상세 뷰**로 자연스럽게 진입하게 구현하라.

이 상세 뷰의 목적은 다음과 같다.

- 사용자가 “주식”이라는 개념을 모르는 상태에서도
- 거시 변수와 주식시장의 내부 구성 요소가 어떻게 연결되는지
- 시각적으로 직관적으로 이해하도록 돕는 것

이 화면은 **실제 투자 분석 도구가 아니라 교육용 인과관계 시뮬레이터**다.

---

## 2. 핵심 UX 개념

### 기본 진입 흐름
- 메인 거시 화면에서 `한국 주식시장` 노드를 클릭
- 현재 거시 상태를 입력값으로 전달
- 주식시장 상세 화면으로 진입
- 상세 화면은 전달받은 거시 상태를 기반으로 초기 렌더링

### 자연스러운 연결
- 사용자는 “거시에서 보던 결과가, 주식시장 내부에서는 어떻게 만들어지는지”를 느껴야 한다
- 따라서 화면 진입 시 갑작스럽게 완전히 다른 제품처럼 보이면 안 된다
- 같은 시각 언어를 유지하라:
  - 어두운 배경
  - 카드형 패널
  - 노드 중심 구조
  - 상승/하락 압력 색상
  - 연결선 강조
  - 우측 설명 패널

---

## 3. 구현 범위

이번 구현은 **한국 주식시장 상세 뷰 1개만** 만든다.

포함 범위:
- 거시 상태 전달
- 주식시장 내부 노드 구조 시각화
- 노드별 상승/하락 압력 계산
- 설명 패널
- 간단한 시계열 주가 반응 표현
- 뒤로가기 / 상위 화면 복귀

제외 범위:
- 실제 주가 데이터 연동
- 뉴스 피드
- 종목 단위 분석
- 호가창, 캔들차트, 기술적 지표
- 포트폴리오 기능

## 4. 상세 화면 개념 모델

주식시장은 아래 요소들로 표현한다.

### 입력: 상위 거시 화면에서 전달받는 상태
- 한국 성장률 `kr_growth`
- 한국 물가 `kr_inflation`
- 한국 기준금리 `kr_rate`
- 원/달러 환율 `usdkrw`
- 미국 기준금리 `us_rate`
- 유가 `oil`
- 외국인 자금 `foreign_flow` (거시 엔진에서 이미 계산된 경우 전달)

### 주식시장 내부 핵심 요소
- 기업 실적 기대
- 할인율 부담
- 외국인 수급
- 국내 유동성
- 투자 심리
- 최종 결과: 한국 주가

### 구조 의도
사용자는 다음 흐름을 이해해야 한다.

- 성장률이 오르면 기업 실적 기대가 좋아진다
- 물가와 금리가 오르면 할인율 부담이 커진다
- 환율이 오르면 외국인 자금 이탈 압력이 커질 수 있다
- 여러 힘이 합쳐져 주가가 결정된다

## 5. 노드 구성

상세 화면은 다음 노드로 구성하라.

### 왼쪽: 거시 입력 노드
- 한국 성장률
- 한국 물가
- 한국 기준금리
- 원/달러 환율
- 미국 기준금리
- 유가

이 노드들은 상세 화면에서 직접 수정 가능하도록 해도 되고,
초기 버전에서는 읽기 전용 + “메인 화면에서 수정” 안내만 있어도 된다.
단, 구현 난이도가 높지 않다면 상세 화면에서도 소폭 조정 가능하게 하라.

### 중앙: 주식시장 내부 노드
- 기업 실적 기대
- 할인율 부담
- 외국인 수급
- 국내 유동성
- 투자 심리

### 오른쪽: 결과 노드
- 한국 주가

## 6. 인과관계 규칙

다음은 교육용 단순화 룰셋이다.  
현실을 100% 복제하려 하지 말고, 방향성과 설명력을 우선하라.

### 6.1 거시 → 내부 요소

- 한국 성장률 상승 → 기업 실적 기대 상승
- 한국 물가 상승 → 할인율 부담 상승
- 한국 기준금리 상승 → 할인율 부담 상승
- 원/달러 환율 상승 → 외국인 수급 악화
- 미국 기준금리 상승 → 외국인 수급 악화
- 유가 상승 → 기업 비용 부담 증가 → 기업 실적 기대 약화
- 유가 상승 → 물가 자극 → 할인율 부담 상승에 간접 기여

### 6.2 내부 요소 → 결과

- 기업 실적 기대 상승 → 한국 주가 상승 압력
- 할인율 부담 상승 → 한국 주가 하락 압력
- 외국인 수급 개선 → 한국 주가 상승 압력
- 국내 유동성 증가 → 한국 주가 상승 압력
- 투자 심리 개선 → 한국 주가 상승 압력

### 6.3 내부 요소 간 연결

- 기업 실적 기대 상승 → 투자 심리 개선
- 외국인 수급 악화 → 투자 심리 악화
- 기준금리 상승 → 국내 유동성 감소
- 성장률 상승 → 국내 유동성 개선에 약한 기여 가능

## 7. 계산 모델

복잡한 경제 모델이 아니라 **설명 가능한 가중치 모델**로 구현하라.

### 방식
각 입력값은 0~100 범위의 상대값이다.  
기준값 50이 중립이다.

각 노드의 delta를 계산하라.

예시:
- 50 기준보다 높으면 양(+) 방향
- 50 기준보다 낮으면 음(-) 방향

### 내부 노드 계산 예시
- 기업 실적 기대 =
  - `+ 0.6 * (kr_growth - 50)`
  - `- 0.2 * (oil - 50)`

- 할인율 부담 =
  - `+ 0.6 * (kr_rate - 50)`
  - `+ 0.3 * (kr_inflation - 50)`

- 외국인 수급 =
  - `- 0.5 * (usdkrw - 50)`
  - `- 0.4 * (us_rate - 50)`

- 국내 유동성 =
  - `- 0.5 * (kr_rate - 50)`
  - `+ 0.2 * (kr_growth - 50)`

- 투자 심리 =
  - `+ 0.3 * 기업 실적 기대`
  - `+ 0.2 * 외국인 수급`
  - `- 0.3 * 할인율 부담`

- 한국 주가 =
  - `+ 0.45 * 기업 실적 기대`
  - `- 0.45 * 할인율 부담`
  - `+ 0.35 * 외국인 수급`
  - `+ 0.25 * 국내 유동성`
  - `+ 0.2 * 투자 심리`

### 정규화
- 모든 결과는 -100 ~ +100 범위로 clamp
- UI에는 다음 상태로 매핑
  - 강한 상승 압력
  - 상승 압력
  - 중립
  - 하락 압력
  - 강한 하락 압력

## 8. 시계열 주가 표현

이 도구는 실제 주가 차트가 아니라  
**“현재 상태가 시간에 따라 어떻게 반영되는지”**를 보여줘야 한다.

### 요구사항
- 오른쪽 결과 영역 아래에 작은 시계열 라인 차트를 표시
- x축은 `즉시 / 단기 / 중기`
- y축은 주가 압력의 변화 방향
- 예:
  - 즉시: 외국인 수급과 심리 반영
  - 단기: 할인율 부담, 유동성 반영
  - 중기: 실적 기대 반영

### 규칙
- 즉시값, 단기값, 중기값을 각각 따로 계산해도 되고
- 하나의 최종 값을 시간차 가중치로 나눠 표현해도 된다
- 중요한 것은 “시간차 반응”을 보여주는 것

## 9. UI/비주얼 요구사항

아름다운 비주얼은 매우 중요하다.  
이 프로젝트는 단순 기능 구현이 아니라 **학습 의욕을 자극하는 시각화 도구**여야 한다.

### 스타일 방향
- 다크 테마 기반
- 미니멀하지만 밀도 있는 정보 디자인
- 블룸/글로우를 약하게 사용
- 노드 카드에 깊이감
- 선은 방향성을 명확하게 드러내야 함

### 색상 원칙
- 상승 압력: 청록/파랑 계열
- 하락 압력: 주황/빨강 계열
- 중립: 회색/보라빛 중성
- 너무 원색적이지 않게 세련되게 조정

### 노드 표현
각 노드는 카드처럼 보이되 아래를 포함하라.
- 제목
- 현재 상태 텍스트
- 작은 방향 아이콘
- 강도 표시 막대 또는 링
- 필요 시 지역 태그 (`KR`, `US`)

### 연결선
- 인과 방향 화살표
- 활성화된 경로는 밝고 굵게
- 비활성 경로는 흐리게
- hover 시 해당 경로 강조

### 모션
- 노드 값이 변할 때 부드럽게 전환
- 연결선 강조 애니메이션
- 주가 라인 차트도 부드럽게 변화
- 과한 모션은 금지

## 10. 우측 설명 패널

상세 화면 오른쪽에는 설명 패널을 두어라.

### 포함 내용
- 현재 주가 압력 요약
- 가장 큰 상승 요인 2개
- 가장 큰 하락 요인 2개
- 즉시 / 단기 / 중기 해석
- “왜 이런 결과가 나왔는가”를 자연어로 설명

### 예시 문장
- “한국 성장률 개선이 기업 실적 기대를 끌어올려 주가에 긍정적으로 작용하고 있습니다.”
- “다만 기준금리 상승으로 할인율 부담이 커져 상승 폭은 제한되고 있습니다.”
- “원/달러 환율 상승은 외국인 수급에 부담을 주며 단기적으로 하락 압력으로 작용합니다.”

이 설명은 룰 기반으로 생성하라.  
LLM 호출 없이 클라이언트 로직으로 충분히 만들 수 있다.

## 11. 상위 화면과의 연결 방식

### 진입 방식
- 거시 화면에서 `한국 주식시장` 노드 클릭
- 현재 macro state를 router state 또는 query-serializable state로 전달
- 상세 화면 진입 시 해당 state로 초기화

### 복귀 방식
- 상단에 `← 거시 화면으로 돌아가기`
- 복귀 시 기존 거시 상태가 유지되어야 함

## 16. 구현 시 유의사항

가장 중요한 것은 **정확성보다 설명력**,  
그리고 **복잡성보다 직관성**이다.

이 프로젝트는 현실 경제를 완벽히 시뮬레이션하는 것이 아니라  
학습자의 머릿속에 올바른 인과 프레임을 심어주는 것이 목적이다.

따라서 구현 중 선택이 필요할 때는 아래 원칙을 따른다.

1. 더 단순한 구조를 선택할 것  
2. 더 아름답고 직관적인 표현을 선택할 것  
3. 결과보다 과정이 보이게 만들 것  
4. 숫자보다 관계를 느끼게 만들 것

이 원칙을 기준으로 완성도 높은 UI와 구조를 구현하라.
