# ePulse 거시 경제 메인 화면에 금(안전자산) 노드 추가 구현 프롬프트

당신은 React 기반 정적 웹앱을 구현하는 시니어 프론트엔드 엔지니어다.  
이 작업은 ePulse의 **거시 경제 메인 인과관계 화면**에 **금(Gold, 안전자산)** 노드를 추가하는 것이다.

이 프로젝트는 실전 투자 도구가 아니라 **경제 인과관계 학습 도구**다.  
정확한 예측보다 **설명력, 직관성, 시각적 명확성**이 우선이다.

---

## 1. 목표

기존 거시 인과관계 화면에 **금(안전자산)** 노드를 추가하여,  
사용자가 아래를 자연스럽게 이해할 수 있게 하라.

- 금은 단순 원자재가 아니라 안전자산적 성격이 있다
- 시장 불안, 물가 상승, 금리 변화, 달러 강세/약세가 금에 영향을 줄 수 있다
- 주식/채권/환율과는 다른 방식으로 반응하는 자산이 존재한다

이 노드는 “안전자산 전체”가 아니라 **교육용 대표 자산으로서 금 하나만** 다룬다.

---

## 2. 왜 추가하는가

현재 거시 메인 화면에는 다음 축이 있다.

- 금리
- 환율
- 채권
- 주식
- 현물(원자재)
- 성장률
- 물가

이 구조만으로도 핵심은 충분하지만,  
**“시장 불안 시 자금이 어디로 피하는가”**라는 중요한 흐름이 빠져 있다.

금 노드 추가의 목적은 다음을 보완하는 것이다.

- 위험 회피 시 자금 이동 방향
- 인플레이션 헤지 자산의 개념
- 금리와 달러 변화에 대한 비주류 반응 자산의 존재

---

## 3. 설계 원칙

### 반드시 지켜야 할 원칙
1. **금은 “현물” 노드 안에 흡수하지 말고 별도 노드로 추가할 것**
2. “은”, “비트코인”, “엔화”, “스위스프랑” 등은 이번 범위에 포함하지 말 것
3. 복잡한 설명보다 **대표적 관계만 보여줄 것**
4. 금은 “예외가 많은 자산”이므로, 과도한 확정 표현을 피하고 “압력” 중심으로 표현할 것

### 핵심 철학
금 노드는 **정답 기계**가 아니라  
“아, 이런 상황에서 금이 관심받을 수 있구나”를 학습시키는 용도다.

---

## 4. 노드 추가 위치

기존 거시 경제 메인 화면의 market 레이어에 다음 노드를 추가하라.

- ID: `gold`
- Label: `금`
- Region: `GL`
- Layer: `market`

시각적으로는 다음 위치가 적절하다.

- `현물(원자재)`와 완전히 겹치지 않게 분리
- `채권`, `주식`, `환율`과의 관계가 보이도록 중앙-우측 market 레이어에 배치
- 너무 구석으로 빼지 말 것
- “시장 불안 시 자금 피난처” 느낌이 자연스럽게 읽히도록 배치할 것

권장 위치:
- `채권`과 `현물` 사이 또는
- `주식` 아래 / `채권` 옆의 독립 market 노드

또는

추천:

- 툴팁/설명에는 `안전자산 성격을 가진 대표 자산`이라고 설명

### 상태 라벨
다른 market 노드와 동일하게 아래 상태 체계를 사용하라.

- 상승 압력
- 중립
- 하락 압력
- 강한 하락 압력

## 6. 금 노드의 핵심 인과관계

MVP에서는 아래 4개 축만 반영하라.

### 6.1 물가 → 금
- 물가 상승 → 금 상승 압력

교육적 의미:
- 금은 인플레이션 헤지 자산으로 자주 인식된다

### 6.2 금리 → 금
- 금리 상승 → 금 하락 압력

교육적 의미:
- 이자가 없는 금은 금리가 높아질수록 상대 매력이 약해질 수 있다

주의:
- 현실적으로는 **실질금리**가 더 중요하지만, MVP에서는 노출하지 말 것
- 설명에는 “일반적으로” 또는 “대체로”라는 뉘앙스를 유지할 것

### 6.3 달러 강세 / 원달러 상승 → 금 하락 압력
- 달러 강세가 커질수록 금 하락 압력

교육적 의미:
- 금과 달러는 종종 반대 방향 압력을 받는다

구현상:
- 메인 화면에 `usd_strength`가 있으면 그것을 직접 사용
- 없다면 `usdkrw_pressure` 또는 `us_rate`에서 간접적으로 유도 가능
- UI에는 “달러 강세 압력”이라는 설명 문구로 표현 가능

### 6.4 위험 회피 / 시장 불안 → 금 상승 압력
- 주식 약세 / 위험 회피 심리 / 시장 불안 → 금 상승 압력

중요:
- 현재 MVP에서 `risk` 노드를 제거했거나 제거 예정이라면,
  **직접 risk 노드를 다시 넣지 말고**
  `주식시장 약세` 또는 `주식 하락 압력`을 약한 대체 신호로 사용해도 된다

권장:
- `kr_stock` 또는 `us_stock` 약세를 간접 신호로 사용
- 예:
  - 미국 주식 하락 압력 → 금 상승 압력 (약한 연결)
  - 한국 주식 하락 압력 → 금 상승 압력 (아주 약한 연결 또는 생략 가능)

## 7. 권장 규칙 정의

다음과 같은 causal rule을 추가하라.

### 필수 규칙

#### r_gold_01
- Source: `kr_inflation`
- Target: `gold`
- Direction: `+`
- Weight: `0.35`
- Lag: `단기`
- Description: 물가 상승은 금에 대한 헤지 수요를 자극할 수 있음

#### r_gold_02
- Source: `kr_rate`
- Target: `gold`
- Direction: `-`
- Weight: `0.35`
- Lag: `단기`
- Description: 금리 상승은 금의 상대 매력을 낮출 수 있음

#### r_gold_03
- Source: `us_rate`
- Target: `gold`
- Direction: `-`
- Weight: `0.30`
- Lag: `즉시`
- Description: 미국 금리 상승은 달러 및 금리 매력을 높여 금에 부담이 될 수 있음

#### r_gold_04
- Source: `usd_strength`
- Target: `gold`
- Direction: `-`
- Weight: `0.40`
- Lag: `즉시`
- Description: 달러 강세는 금 가격에 하락 압력으로 작용할 수 있음

### 선택 규칙

#### r_gold_05
- Source: `us_stock`
- Target: `gold`
- Direction: `-`
- Weight: `0.20`
- Lag: `단기`
- Description: 미국 주식이 강하면 안전자산 선호는 약해질 수 있음

또는 보다 직관적으로 내부적으로는 다음처럼 처리 가능:
- `us_stock` 하락 압력일수록 `gold` 상승 압력

구현 난이도에 따라 둘 중 하나를 선택하라.  
단, 학습 설명은 반드시 자연스럽게 보여야 한다.

## 8. 계산 철학

금은 다른 자산보다 예외가 많기 때문에  
**지나치게 강한 단선형 규칙으로 만들지 말 것**.

### 권장 계산 성격
- 금은 여러 요소의 “균형”으로 결정되는 노드로 구현
- 하나의 입력으로 극단적으로 튀지 않게 가중치 균형 유지
- `clamp(-100, 100)` 사용
- 설명 패널에서는 “상승/하락 압력” 중심으로 표현

예시:
goldDelta =
- `+ 0.35 * inflationEffect`
- `- 0.35 * krRateEffect`
- `- 0.30 * usRateEffect`
- `- 0.40 * usdStrengthEffect`
- `+ 0.20 * safeHavenEffect`

## 9. 메인 화면 설명 패널 업데이트

금 노드가 추가되면 우측 설명 패널 또는 노드 클릭 설명에 아래 유형의 문장을 생성하라.

### 예시 문장
- “물가 상승은 금에 대한 헤지 수요를 자극해 상승 압력으로 작용할 수 있습니다.”
- “반면 금리 상승은 이자가 없는 금의 상대 매력을 낮춰 하락 압력으로 이어질 수 있습니다.”
- “달러 강세가 커질수록 금은 상대적으로 부담을 받을 수 있습니다.”
- “주식시장 불안이 커질 경우 금이 안전자산으로 선호될 수 있습니다.”

### 주의사항
- “항상”, “반드시” 같은 단정 표현 금지
- “일반적으로”, “대체로”, “압력”, “경향” 같은 표현 사용

---

## 10. 시각적 표현

### 색상
금 노드는 기존 상승/하락 색 체계를 따르되,  
노드 자체의 기본 액센트는 은은한 금색/앰버 계열을 사용할 수 있다.

단:
- 상승/하락 상태 색상 체계를 깨면 안 됨
- 기본 테두리나 아이콘에만 금색 포인트 사용 가능
- 상태 표현은 전체 앱과 동일한 색 규칙 유지

### 아이콘
가능하면 금 관련 아이콘을 사용:
- 원형 메달
- 금괴 느낌의 단순 아이콘
- 너무 사실적이거나 게임 UI처럼 보이지 않게 할 것

### 강조
금 노드 클릭 시 아래 연결 경로가 잘 보이게 하라.

- 물가 → 금
- 금리 → 금
- 달러 강세 → 금
- 주식 약세/시장 불안 → 금

---

## 11. 레이아웃 복잡도 관리

금 노드를 추가하더라도 그래프가 과밀해지면 안 된다.

### 반드시 지킬 것
- 연결선 수를 최소화
- 보조 관계는 약하게 표현
- 핵심 관계만 밝고 선명하게 표시
- hover / click 시 관련 경로만 강조되게 만들 것

### 연결선 우선순위
1. 달러 강세 → 금
2. 금리 → 금
3. 물가 → 금
4. 주식 약세 / 불안 → 금

---

## 12. 상세 뷰 연결 가능성 고려

이번 작업은 메인 노드 추가가 목적이지만,  
추후 `금` 노드를 클릭했을 때 **금 상세 학습 뷰**로 진입할 수 있도록 구조를 열어두어라.

즉:
- `gold` 노드는 다른 market 노드처럼 clickable 해야 한다
- 현재는 상세 뷰가 없어도 route hook 또는 placeholder 구조를 열어둘 것

예:
- `/detail/gold`

## 13. RULESET 반영 지침

현재 RULESET 문서가 있다면 다음을 반영하라.

### Derived 변수에 추가
- `gold | 금 | GL | market`

### Rules에 추가
위에서 제시한 `r_gold_01` ~ `r_gold_04`를 우선 추가

### Contradiction Rules
금 노드는 contradiction rule을 반드시 추가할 필요는 없다.
MVP에서는 생략 가능하다.

다만 원한다면 아래 정도는 가능하다.

- 고금리 + 강한 금 상승 압력이 동시에 pinned된 경우 warning
- 달러 강세 + 강한 금 상승 압력이 동시에 pinned된 경우 warning

하지만 이번 범위에서는 필수가 아니다.

## 17. 설계 원칙

선택이 필요할 때는 아래를 우선하라.

1. 정확성보다 설명력
2. 복잡성보다 직관성
3. 정보량보다 시각적 명확성
4. 수치 예측보다 인과관계 학습
5. 확장성보다 현재 구조의 깔끔함

이 원칙을 기준으로 금(안전자산) 노드를 메인 거시 경제 화면에 자연스럽고 아름답게 통합하라.
