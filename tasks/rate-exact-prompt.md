# ePulse 금리 상세 학습 뷰 구현 프롬프트

당신은 React 기반 정적 웹앱을 구현하는 시니어 프론트엔드 엔지니어다.  
이 작업은 ePulse의 **금리 상세 학습 뷰**를 구현하는 것이다.

---

## 1. 목표

거시 인과관계 메인 화면에서 사용자가 **금리 노드**를 클릭하면,  
현재 거시 상태를 전달받아 **금리가 어떻게 결정되고 왜 시장금리와 기준금리가 다르게 움직일 수 있는지**를 직관적으로 이해할 수 있는 상세 화면을 구현하라.

이 화면은:
- 금리 개념을 모르는 사용자도 이해 가능해야 하며
- 물가, 성장률, 환율, 유가가 금리에 어떻게 연결되는지 보여야 하고
- 기준금리와 시장금리의 차이를 교육적으로 설명해야 한다

이 화면은 **실전 투자 도구가 아니라 학습용 인과관계 시뮬레이터**다.

---

## 2. 핵심 UX 개념

사용자는 이 화면을 보고 아래를 이해해야 한다.

- 금리는 왜 오르고 내리는가
- 기준금리와 시장금리는 무엇이 다른가
- 왜 시장금리가 먼저 움직일 수 있는가
- 금리가 채권/주식/환율에 중요한 이유가 무엇인가

핵심은 **금리를 숫자가 아니라 “결정 구조”로 보이게 하는 것**이다.

---

## 3. 상위 화면과의 연결

### 진입 방식
- 거시 화면에서 `한국 기준금리` 또는 `한국 시장금리` 관련 노드 클릭
- 현재 macro state를 상세 화면으로 전달
- 전달 방식은 router state 또는 query-serializable state 사용

### 복귀 방식
- 상단에 `← 거시 화면으로 돌아가기`
- 복귀 시 원래 거시 상태 유지

### 자연스러운 연결
- 메인 화면과 동일한 시각 언어 유지
- 다크 테마, 카드 기반 노드, 연결선 강조, 설명 패널 유지

- 시장 기대

### 3층: 결과
- 한국 기준금리

## 6. 핵심 인과관계

### 6.1 입력 → 내부 구조

- 한국 물가 상승 → 인플레이션 압력 상승
- 유가 상승 → 인플레이션 압력 상승
- 원/달러 환율 상승 → 수입물가 압력 상승 → 인플레이션 압력 상승
- 한국 성장률 상승 → 경기 압력 상승
- 미국 기준금리 상승 → 시장 기대를 통해 한국 시장금리 상승 압력

### 6.2 내부 구조 → 정책/시장

- 인플레이션 압력 상승 → 중앙은행 스탠스 매파화
- 경기 압력 상승 → 중앙은행 스탠스 긴축 성향 강화
- 경기 둔화 → 중앙은행 스탠스 완화 성향 강화

- 인플레이션 압력 상승 → 시장 기대에서 금리 인상 기대 강화
- 미국 기준금리 상승 → 시장 기대에서 금리 인상/고금리 지속 기대 강화

### 6.3 정책/시장 → 결과

- 중앙은행 스탠스 매파화 → 기준금리 상승 압력
- 중앙은행 스탠스 완화화 → 기준금리 하락 압력

- 시장 기대가 매파적일수록 → 시장금리 상승 압력
- 시장 기대가 완화적일수록 → 시장금리 하락 압력

### 6.4 기준금리와 시장금리 관계

- 기준금리 변화는 시장금리에 영향을 준다
- 그러나 시장금리는 미래 기대를 먼저 반영할 수 있다
- 따라서 시장금리가 기준금리보다 먼저 움직이는 표현이 가능해야 한다

## 7. 계산 모델

현실을 정밀 복제하지 말고, **설명 가능한 단순 가중치 모델**을 사용하라.

모든 입력값은 0~100 범위이며, 50이 중립이다.

### 7.1 1차 내부 노드 계산

#### 인플레이션 압력
inflationPressure =
- `+ 0.6 * (kr_inflation - 50)`
- `+ 0.3 * (oil - 50)`
- `+ 0.25 * (usdkrw - 50)`

#### 경기 압력
growthPressure =
- `+ 0.6 * (kr_growth - 50)`

### 7.2 2차 내부 노드 계산

#### 중앙은행 스탠스
centralBankStance =
- `+ 0.55 * inflationPressure`
- `+ 0.35 * growthPressure`

해석:
- 값이 높을수록 긴축/매파
- 값이 낮을수록 완화/비둘기

#### 시장 기대
marketExpectation =
- `+ 0.45 * inflationPressure`
- `+ 0.25 * growthPressure`
- `+ 0.35 * (us_rate - 50)`
- `+ 0.2 * centralBankStance`

해석:
- 값이 높을수록 금리 인상 기대
- 값이 낮을수록 금리 인하 기대

### 7.3 결과 노드 계산

#### 한국 기준금리
krBaseRateResult =
- `+ 0.8 * centralBankStance`

#### 한국 시장금리
krMarketRateResult =
- `+ 0.55 * marketExpectation`
- `+ 0.25 * krBaseRateResult`

모든 결과는 `-100 ~ +100` 범위로 clamp 하라.

## 9. 핵심 시각화 요구사항

### 가장 중요한 표현
이 화면은 반드시 아래를 보여줘야 한다.

- 기준금리와 시장금리가 **같지 않을 수 있음**

## 10. UI 레이아웃

### 좌측 패널
입력 요인 카드 목록:
- 한국 물가
- 한국 성장률
- 원/달러 환율
- 유가
- 미국 기준금리

각 카드에는:
- 제목
- 현재 상태
- 작은 방향 아이콘
- 값 강도 표시

읽기 전용으로 시작해도 되지만,
구현 난이도가 크지 않다면 작은 슬라이더를 넣어 로컬 조정 가능하게 하라.

### 중앙 캔버스
노드 기반 구조 시각화:
- 입력 노드 → 내부 구조 노드 → 결과 노드
- 활성 경로는 밝고 굵게
- 약한 경로는 흐리게
- hover 시 관련 경로만 강조

### 오른쪽 패널
설명 패널:
- 현재 금리 구조 요약
- 기준금리 해석
- 시장금리 해석
- 왜 둘이 다를 수 있는지 설명
- 즉시 / 단기 / 중기 흐름 설명

## 11. 시계열 표현

하단 또는 오른쪽 하단에 작은 라인 차트 추가

### 목적
- “시간차 반응”을 보여주는 것

### x축
- 즉시
- 단기
- 중기

### 두 개의 선
- 시장금리
- 기준금리

### 의도
- 시장금리 선이 먼저 반응
- 기준금리 선은 더 늦게, 더 완만하게 반응

### 예시 해석
- 즉시: 시장 기대 반영 → 시장금리 먼저 움직임
- 단기: 중앙은행 스탠스 강화
- 중기: 기준금리 조정 반영

## 12. 설명 패널 문장 생성 규칙

설명은 룰 기반 자연어 생성으로 구현하라.  
LLM 호출은 사용하지 않는다.

### 반드시 포함할 설명
- 현재 기준금리 압력
- 현재 시장금리 압력
- 둘의 차이 설명
- 가장 큰 상승 요인
- 가장 큰 하락/완화 요인
- 시간 흐름 해석

### 예시 문장
- “한국 물가와 유가 상승이 인플레이션 압력을 키우며 중앙은행의 긴축 성향을 강화하고 있습니다.”
- “시장에서는 이미 금리 인상 기대가 반영되며 시장금리가 기준금리보다 먼저 상승하고 있습니다.”
- “기준금리는 중앙은행의 실제 결정값이고, 시장금리는 미래 기대가 먼저 반영된 결과입니다.”
- “성장률 둔화가 뚜렷하다면 긴축 강도는 일부 완화될 수 있습니다.”

## 13. 시각 스타일

아름다운 비주얼은 매우 중요하다.  
이 도구는 단순 교육용 화면이 아니라 **탐색하고 싶은 감각적인 학습 도구**여야 한다.

### 스타일 가이드
- 다크 배경
- 깊이감 있는 카드
- 선명하지만 과하지 않은 글로우
- 부드러운 애니메이션
- 미니멀한 정보 밀도

### 색상 원칙
- 상승/긴축/인상 기대: 청록~파랑 계열
- 하락/완화/인하 기대: 주황~빨강 계열
- 중립: 회색/중성톤
- 같은 의미의 색상은 앱 전체에서 일관되게 유지

### 노드 카드
각 노드는 다음을 포함:
- 제목
- 상태 라벨
- 아이콘
- 미니 강도 표시 바 또는 원형 인디케이터
- 지역 태그 (KR / US / GL 필요 시)

## 14. 인터랙션

### hover
- 노드 hover 시 관련 경로 강조
- 우측 설명 패널에 해당 노드 설명 강조 가능

### click
- 결과 노드 클릭 시 관련 원인 경로 전체 강조
- 예: `한국 시장금리` 클릭 시
  - 시장 기대
  - 미국 금리
  - 물가
  - 환율
  경로가 더 밝게 보이도록 처리

### animation
- 값 변화 시 노드 상태 부드럽게 전환
- 연결선 강조 시 은은한 흐름 효과
- 라인 차트 부드럽게 재계산

## 19. 설계 원칙

선택이 필요할 때는 아래를 우선하라.

1. 정확성보다 설명력
2. 복잡성보다 직관성
3. 숫자보다 관계
4. 결과보다 과정
5. 정보량보다 시각적 명확성

이 원칙을 기준으로 완성도 높은 UI와 교육용 인과관계 경험을 구현하라.
